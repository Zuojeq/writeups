================


_John Hammond_ | _Wednesday, December 30th, 2015_ 


> Mommy! what is a file descriptor in Linux?

> ssh fd@pwnable.kr -p2222 (pw:guest)


----------

You can log into the server with [`ssh`][ssh] and explore the files at hand with the `ls` command. We have [source code](fd.c) available to us, so let's take a look at it.

``` c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
char buf[32];
int main(int argc, char* argv[], char* envp[]){
	if(argc<2){
		printf("pass argv[1] a number\n");
		return 0;
	}
	int fd = atoi( argv[1] ) - 0x1234;
	int len = 0;
	len = read(fd, buf, 32);
	if(!strcmp("LETMEWIN\n", buf)){
		printf("good job :)\n");
		system("/bin/cat flag");
		exit(0);
	}
	printf("learn about Linux file IO\n");
	return 0;

}
```

So the program clearly takes an [argument], and it looks like it opens up a file based off of a number, with our supplied [argument] having a [hex] value `0x1234` subtracted from it. Well, what is that [hex] value in decimal? We can just throw it at a [Python] interpreter and we'll get it real quick:

``` python
>>> 0x1234
4660
```

Okay, so we've got this number `4660`, but how does that help us? It looks like if the program reads from the file this string `LETMEWIN\n`, we get our flag. But we need a file that is, like, based off a number... huh?

If we take the hint that the challenge prompts us to, we should research [file descriptors]. 

Immediately, we should see this kind of references [standard streams], -- because remember, __In [Linux], _everything_ is a file!__

So we've got a [file descriptor] `0` that is [standard input]... can we use that? 

Remember the argument that we pass the program will have `4660` subtracted from it, so to get `0`, we simply just have to pass in `4660`!

```
$ ./fd 4660

```

Nothing happens.

Well, _something_ happens! The program is now waiting for our [_standard input_], right? So let's type in what it wants to hear: that `LETMEWIN` string.

And we get our flag!

Nice. Since this is just [standard input], we could just as easily [pipe] that string into `echo` and just have the program spew out our flag essentially the same way.

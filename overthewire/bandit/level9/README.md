__Bandit :: Level 9__
================


_John Hammond_ | _Wednesday, January 13th, 2016_ 


> The password for the next level is stored in the file data.txt in one of the few human-readable strings, beginning with several ‘=’ characters.


----------


Log in as usual to the account with the password you retrieved from the previous level.

```
ssh bandit9@bandit.labs.overthewire.org
```

The `data.txt` file again is in the [home directory]. When we [`cat`][cat] that out, it's clear we have to do something to automate this and find our flag.

Since it has a bunch of [binary data], we can't really [`grep`][grep] anything out of. At all.

```
$ cat data.txt | grep "anything
Binary file (standard input) matches
```

Well, great. The challenge prompt tells us that it is in a human readable string, however -- so we can scrape all of those out, can't we?

That is where the [`strings`][strings] command comes in nicely. [`strings`][strings] was listed in the "commands you may need to solve this level," so I would hope that you have taken a look at it.

Simply, [`strings`][strings] will grab all the human readable strings out of a file (or output). So, let's try and feed it the `data.txt` file!

```
$ cat data.txt | strings
```

If you even [pipe] this through [`less`][less], you can peruse through it all on your own. But, [`strings`][strings] will grab some other short garbage (just because that data may be representing something in the [binary] junk), so disregard that. You can actually try and help optimize it with some arguments to [`strings`][strings]. I really recommend you stroll through it's [`man`][man] page and play with it a bit.

Regardless, we could now [`grep`][grep] pretty easily for any of those `=` equals signs -- if you didn't already see the password with just [`strings`][strings] by itself.

```
$ cat data.txt |strings|grep "="
I========== the6
========== password
========== ism
========== truKLdjsbJ5g7yyJ2X2R0o3a5HQJFuLk
```

Got our next password!

__The password for bandit10 is `truKLdjsbJ5g7yyJ2X2R0o3a5HQJFuLk`.__

__Leviathan :: Level 4__
================


_John Hammond_ | _Saturday, November 7th, 2015_ 


> There is no information for this level, intentionally.


----------


Start by [`ssh`][ssh]ing into the level. Supply the password you retrieved in the previous level. 

```
ssh leviathan4@leviathan.labs.overthewire.org
```

If you run the [`ls`][ls] command, you'll see there is nothing actually in this directory. You'll have to use the same tactic as in a previous level, where you run `ls -a` to show the hidden files. That shows you there actually is a `.trash` directory you can `cd` into.

Inside that directory, you'll see a `bin` file. Let's try and execute this. 

```
leviathan4@melinda:~/.trash$ ./bin 
01010100 01101001 01110100 01101000 00110100 01100011 01101111 01101011 01100101 01101001 00001010
```

I wonder what this is. Let's try our usual [`ltrace`][ltrace] strategy and see what this program actually does. 

```
leviathan4@melinda:~/.trash$ ltrace ./bin
__libc_start_main(0x80484cd, 1, 0xffffd784, 0x80485c0 <unfinished ...>
fopen("/etc/leviathan_pass/leviathan5", "r")        = 0
+++ exited (status 255) +++
```

Oh! It looks like it opens up the password file for the next level. So this output must be the password for leviathan5, awesome! But it looks like it's all in binary...

So I honestly don't entirely how to get a plaintext conversion of this within [Python] just yet. If you happen to figure it out, let me know. 

You _can_ however, use an [online converter](http://www.binaryhexconverter.com/binary-to-ascii-text-converter) to get this into plaintext.

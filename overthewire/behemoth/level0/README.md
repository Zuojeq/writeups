__Behemoth :: Level 0__
================


_John Hammond_ | _Saturday, November 7th, 2015_ 


> There is no information for this level, intentionally.


----------


Start by [`ssh`][ssh]ing into the level. The password is given to us in the initial description of the wargame, `behemoth0`.

```
ssh behemoth0@behemoth.labs.overthewire.org
```


If you run the [`ls`][ls] command, you'll see there is nothing in this directory (`ls -a` will tell you the same thing).

Don't forget that initial prompt of the wargame tells you files for each level are found under `/behemoth/`. So let's `cd` to there.

We only have the permissions to run just one file; `behemoth0`. Let's run it.

We're prompted with a password, and if we enter anything we're booted out of the program. 

```
behemoth0@melinda:~$ cd /behemoth
behemoth0@melinda:/behemoth$ ls
behemoth0  behemoth2  behemoth4  behemoth6         behemoth7
behemoth1  behemoth3  behemoth5  behemoth6_reader
behemoth0@melinda:/behemoth$ ./behemoth0 
Password: anything
Access denied..
```

Fire up [`ltrace`][ltrace] to see what this actually does.

```
__libc_start_main(0x80485a2, 1, 0xffffd794, 0x8048690 <unfinished ...>
printf("Password: ")                              = 10
__isoc99_scanf(0x804876c, 0xffffd6ab, 0xffffd6a0, 0x80482d2Password: anything
) = 1
strlen("OK^GSYBEX^Y")                             = 11
strcmp("anything", "eatmyshorts")                 = -1
puts("Access denied.."Access denied..
)                           = 16
+++ exited (status 0) +++
```

Hmm, it looks like it is comparing what we enter with "eatmyshorts". Let's try that as the password, outside of [`ltrace`][ltrace].

Looks like we get a shell! Passwords are stored in the same place as [Leviathan], just under it's own corresponding directory (`/etc/behemoth_pass/behemoth#`)
